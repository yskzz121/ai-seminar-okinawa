💻 知識の底上げ ⭐ 必修

CLIって何？

Claude Codeが動く「土俵」を5分で理解する

⌨️

Command Line Interface（コマンドラインインターフェース）

CLIとは、キーボードから文字を打ってPCに指示を出す画面のこと。マウスも、メニューも、ボタンも使いません。
画面に $ や > が光って、そこに打ち込むだけ。

経営者向けのメタファー

🏢

GUI = 受付カウンター

用意されたメニューから選ぶ。誰でも使えて安心だが、できることはメニューに載ってる範囲だけ。

☎️

CLI = 社長室の直通ダイヤル

相手に直接指示を出す。自由度は無限で「何でも頼める」。ただしどう頼むか知っておく必要がある。

GUI vs CLI

項目GUI（普段のPC）CLI（Claude Codeの舞台）
操作方法マウスでクリック文字を打つ
できること△ ボタンの範囲内◎ ほぼ何でも可
速度△ クリック回数分◎ 一行で完結
自動化✕ 難しい◎ 得意
例Excel, Word, Finderターミナル, Claude Code
難しそうに見えるCLIも
Claude Codeなら日本語で話しかけるだけ。
つまり、CLIの「何でもできる強さ」を、コマンドを覚えずに使えるというのがClaude Codeの革命です。コマンドは全部AIが代わりに打ってくれます。

経営者が覚えておく4つの特徴

🤖

自律的に動く

指示1つで必要な手順を自分で判断。「売上レポート作って」で何を読んで何を計算してどう出力するか組み立ててくれる。

📁

PCの中全体を見る

フォルダ内の数百ファイルを丸ごと把握。「あのファイルに書いてある住所」レベルの質問でも、自分で探して答える。

複数工程をつなぐ

「読む→計算する→書き出す→送る」を1指示で完遂。ChatGPTのように「コピペして」「次はこれ」と誘導する必要なし。

🛡️

暴走しない設計

ファイル削除・送信など危ない操作の前は必ず確認。「これやっていい？」と聞いてくれる。怖がらなくて大丈夫。

MCPってなに？

Claudeに「社外ツールの名刺」を渡す仕組み ― 今夜は概念だけ押さえればOK

🗝️

MCP = 「名刺と鍵」

MCP（Model Context Protocol）は、ClaudeにGmailやGoogleカレンダー、Slack、データベースなどの外部ツールを触る権限を渡す仕組みです。

新入社員に「会社のメールアカウントの名刺」「銀行口座の鍵」を渡すイメージ。渡した分だけ仕事の範囲が広がります。

1

Node.js をインストール

所要時間 約10分ダウンロード 約60MB再起動不要

Node.js（ノードジェイエス）とは、Claude Codeが動くために必要な「エンジン」です。車にガソリンが要るのと同じ。一度入れたら今後も使い回せます。

  1. ブラウザで https://nodejs.org/ を開く
  2. 真ん中の大きな緑ボタン「LTS」（推奨版）をクリック → .pkg ファイルがダウンロードされる
  3. ダウンロードフォルダの node-v20.x.x.pkg をダブルクリック
  4. インストーラーに従って「続ける」を連打。途中でPCのパスワードを聞かれるので入力
  5. 「インストールが完了しました」が出たら閉じる

確認方法：ターミナルで下のコマンドを打つ（ターミナルはStep2で開きます）

Terminal
$ node --version
v20.11.1 ← この形（v + 数字）が出ればOK
  1. ブラウザで https://nodejs.org/ を開く
  2. 真ん中の大きな緑ボタン「LTS」（推奨版）をクリック → .msi ファイルがダウンロード
  3. ダウンロードフォルダの node-v20.x.x-x64.msi をダブルクリック
  4. セキュリティ警告が出たら「実行」。「Next」を連打で基本設定でOK
  5. 途中「Automatically install tools...」のチェックは入れずに進めて大丈夫（必要なら後で）
  6. 最後に「Finish」でインストール完了

確認方法：PowerShell で下のコマンドを打つ

PowerShell
PS> node --version
v20.11.1 ← この形（v + 数字）が出ればOK
✅ ここまでのゴール node --version でバージョンが表示される。表示されたら次へ。
⚠️ 詰まったら ダウンロードが始まらない→別ブラウザで再挑戦／command not foundと出る→Node.jsインストール失敗しているので再インストール／それでもダメな時は挙手

ターミナル（黒い画面）を開く

所要時間 1分最初だけドキッとする画面

Claude Codeを動かす舞台です。怖く見えますが、映画の「社長直通回線」くらいのものだと思ってください。

  3. 「Windows PowerShell」をクリックして開く

⚠️ 権限問題対策 npm install -g がうまくいかない人は、PowerShellを右クリック→「管理者として実行」で開き直してください。
✅ ここまでのゴール 黒い（または青／白）画面が開いて、打ち込みを待っている状態。

🎯 FIRST RUN ⭐ 必修

最初の一言と覚えておきたい操作

動いたあと、やっていい「最初の5分の遊び方」

覚えるのは4つだけ

⌨️

指示を出す

日本語で文章を打って Enter。敬語でも口語でもOK。長文でもOK、短文でもOK。

⏸️

止めたい時

Esc キーを押すと作業を中断。怖くなったらEsc。暴走してもこれで止まる。

🚪

終わる時

/exit と打って Enter、または Ctrl+C を2回。次回は claude で再起動。

❓

困ったら

/help で全コマンド一覧。Claude自身に「これ何ができる？」と聞くのもアリ。

経営者が最初にハマる3つのワナ

  • ✕
    ChatGPTの癖でコピペしようとする
    Claude Codeは自分でファイル書けます。コピペ不要。「○○というファイルを作って」と指示するだけ。
  • ✕
    1回で完璧を求めて詰め込みすぎる
    「まず〇〇」「次に△△」と分けて頼む方が結果が安定します。人間の部下と同じ。
  • ✕
    勝手に動くのを怖がって止めすぎる
    危ない操作は自動で確認してくれます。気軽に任せて大丈夫。見守る胆力がいちばん効きます。
鉄則：今夜試す時、間違いを恐れず「ダメだったら Esc」で押し切る。壊れるファイルは練習フォルダの中だけ。

サブエージェントとは

Claudeの中に「専門の部下」を雇う仕組み ― 今夜は概念だけ

🧑‍💼 → 🧑‍💼🧑‍💼🧑‍💼

社長（メインClaude）が、部門長（サブエージェント）に仕事を振る

社長1人ですべてやるのでなく、経理部長・営業部長・法務部長のような専門の部下を持ち、それぞれに得意分野を任せる仕組み。最大7人同時に動けるので、調べもの・分析・検証を並列で片付けられます。

⚡

並列で高速化

3社の競合調査を同時に回せば、1件ずつの3分の1の時間で終わる。

🎯

専門性を上げる

「セキュリティ専用」「数値分析専用」など、用途特化の部下を作り置きできる。

♻️

一度作れば使い回せる

定義ファイル1つ書けば、他のプロジェクトでも同じ部下が使える。ナレッジが資産化。

今夜の立ち位置：サブエージェントは、基本操作に慣れてから手を出す応用機能です。「Claude Codeには、部下を増やせる仕組みまである」とだけ記憶していれば十分。次回以降の勉強会で扱います。

Claude Codeに至る70年の歴史

ターミナル・コーディング・AIの3本の川が合流した到達点がClaude Code

時代ターミナルコーディング意味（経営者視点）
1950-60sテレタイプ端末（紙に印字）パンチカード・機械語人が機械の言語に合わせていた時代
1970-80sVT100ビデオ端末（画面登場）FORTRAN・COBOL・C言語専門家が増え、産業になっていく
1980-90sMS-DOS・BashTurbo Pascal・Visual Studio統合開発環境(IDE)登場。生産性が飛躍
2000-10siTerm2・Windows TerminalVS Code・GitHub・クラウドネットでの共同作業が常識に
2021-24Warp（AI統合）Copilot・ChatGPT・CursorAIがコードを書き始める
2025-Claude Code自律型エージェントAIが仕事を任されるレベルに到達
70年かけて「人が機械の言語を学ぶ」時代から「機械が人の言葉を理解する」時代になりました。これが、今夜インストールする Claude Code の立ち位置です。
