💻 BASICS

CLIとは何か

コマンドラインインターフェース ― コンピュータとの「対話」の原点

⌨️
💻

Command Line Interface

CLIとは、キーボードからテキストコマンドを入力して、コンピュータに指示を出すインターフェースのこと。
画面に表示される$や>のあとにコマンドを打ち込み、Enterキーで実行します。

分かりやすい例え

🍽️

GUI = レストランのメニュー表

写真付きメニューから料理を選ぶ。何があるか一目で分かるが、メニューにない料理は注文できない。

📋 👆
👨‍🍳

CLI = シェフへの直接オーダー

素材と調理法を直接伝える。自由度は無限だが、料理の知識（コマンドの知識）が必要。

💬 🔥

GUI vs CLI ― 何が違う？

項目GUI（普段使い）CLI（プロ仕様）

処理速度△ 画面描画で遅い◎ 高速（描画不要）
自動化✕ 困難◎ スクリプトで容易

Claude CodeがCLIツールである理由：ターミナルは開発の実務（デプロイ、データベース操作、バージョン管理）が集約される場所。そこにAIを直接統合することで、ツールを切り替える手間なく、AIが自律的に作業を完遂できるのです。

CLIは1960年代にコンピュータと人間が対話する唯一の手段として誕生しました。1984年にMacintoshがGUIを普及させた後も、サーバー管理やソフトウェア開発の現場ではCLIが不可欠でした。

2025年、Claude Codeの登場により、CLIの役割は大きく変わりました。かつては「コマンドを暗記しないと使えない」のが障壁でしたが、今は自然言語（日本語）で指示するだけでAIがコマンドを実行してくれます。CLIの強みを維持しながら、学習コストを劇的に下げた ―― それがClaude Codeの革新です。

🖥️ TERMINAL HISTORY

ターミナルの変遷

60年以上の進化 ― 物理端末からAI統合ターミナルへ

🖥️
POINT

「ターミナル」とは、コンピュータと人間の接点（窓口）のこと。銀行窓口のようなもので、時代と共にその「窓口」の形が進化してきました。

1960年代

テレタイプ端末（TTY）

コンピュータとの最初の対話手段。紙テープとプリンターで入出力を行い、画面は存在しなかった。電信用タイプライターをコンピュータに接続したもので、現在Linuxで使われる「TTY」という用語はここに由来します。

原点

代表的な端末は ASR-33 Teletype（1963年）。1分間に10文字しか印字できませんでした。プログラムの結果は紙に印刷され、プログラマーは印刷結果を見てデバッグしていました。

1970〜80年代

ビデオ端末（VT100）

紙の代わりにブラウン管画面に文字を表示。DEC社の VT100（1978年）が事実上の業界標準となり、現在のターミナルエミュレータの基礎に。

画面の登場

VT100はANSIエスケープコードを初めてサポート。文字の色変更やカーソル移動が可能になりました。DECはVTシリーズで累計600万台以上を販売。同時期にBourne Shell（sh, 1979年）が登場し、シェルスクリプトの基礎が築かれました。

1980〜90年代

PC上のターミナルエミュレータ

IBM PCの普及でMS-DOSがPCのCLIに。1989年にはBashが誕生し、Linuxの標準シェルとして今も現役。

パーソナルに

MS-DOS（1981年〜）ではコマンドプロンプトが標準。xterm（1984年）はX Window System用のターミナルエミュレータで、Unix系OSの標準に。PuTTY（1999年）はSSH接続用クライアントとして、サーバー管理の必需品になりました。

2000年代

macOS Terminal・PowerShell

macOSにターミナルが標準搭載。MicrosoftはPowerShell（2006年）を発表。従来のテキスト処理からオブジェクト指向の革新的シェルへ。

モダン化

macOS Terminal（当時Mac OS X）は内部でBashシェルを採用（後にzshに変更）。PowerShellはJeffrey Snoverが2002年の「Monad Manifesto」で構想を発表し、4年かけて実現。テキストではなくオブジェクトを扱う設計が画期的でした。

2010年代

モダンターミナルの台頭

iTerm2（macOS, 2010年〜）が開発者の定番に。Windows Terminal（2019年）はGPU加速、タブ、分割ペインを搭載しモダン化。

高機能化

iTerm2は分割ペイン、検索、プロファイル管理などmacOS標準Terminalを大幅に超える機能を提供。Hyper（2017年, Vercel開発）はWeb技術ベースのクロスプラットフォーム端末。Alacritty（2017年〜）はRust製で高速描画が特徴です。

2020年代〜

AI統合ターミナルの時代

Warp（2022年〜）が自然言語でのコマンド生成を実現。そしてClaude Code（2025年〜）がターミナル上でAIが自律的にコーディングする新パラダイムを創出。

AI融合

WarpはRust製の「エージェント型開発環境」で、各コマンドを「ブロック」として表示する革新的UIを採用。Ghostty（2024年〜）もモダンアーキテクチャで注目を集めています。Claude Codeは「ターミナル内で動くAIコーディングエージェント」として、ターミナルの概念そのものを再定義しました。

60年間で「紙に印字するタイプライター」→「画面付き端末」→「PC上のソフト」→「AI搭載の知的端末」へ。ターミナルは常に人間とコンピュータの最も効率的な接点であり続けています。
📝 CODING HISTORY

コーディングの変遷

パンチカードからAIエージェントまで ― 70年の進化

📝
1950年代

パンチカード・機械語

カードに穴を開けてプログラムを記述し、コンピュータに読み込ませる。1行間違えると最初からやり直し。デバッグという概念がほぼ存在しなかった時代。

手作業

パンチカードは約90年間にわたりエンジニアとマシンのコミュニケーション手段として使われました。1つのタスクに数日かかることもあり、カードの角が少しでも曲がると使えなくなるリスクがありました。アセンブリ言語が登場し、機械語（0と1の列）を人間が読める記号に置き換え始めました。

1960〜70年代

高級言語の登場（FORTRAN, COBOL, C言語）

FORTRAN（科学技術計算向け）、COBOL（ビジネス向け）が「プログラミングの民主化」の先駆けに。1972年にはC言語が誕生し、現在も使われ続けている。

言語の誕生

テキストエディタ（ed, vi）でソースコードを書き、コンパイラを別途実行する「編集 → コンパイル → 実行」という分離されたワークフローが確立されました。COBOLは英語に近い構文で書けたため、プログラミングの敷居を大きく下げました。

1980年代

IDE（統合開発環境）の誕生

Turbo Pascal（1983年, Borland）がIDE革命の起点。エディタ、コンパイラ、デバッガを1つのプログラムに統合し、開発生産性が劇的に向上した最初の転換点。

統合の始まり

それまでは「エディタで書く → 保存 → 別のツールでコンパイル → 別のツールでデバッグ」と、何度もツールを切り替えていました。Turbo Pascalはこれを一画面で完結させ、プログラマーの効率を飛躍的に高めました。

1990年代

Visual Studio・Eclipse

MicrosoftのVisual Studio（1997年）が業界初のIntelliSense（インテリジェントコード補完）を搭載。入力中にコードを提案する ―― 現在のAI補完の原型がここに。

コード補完の原型

Visual Basic（1991年）はドラッグ&ドロップでUIを構築できるGUIビルダーを統合。Eclipse（2001年）はオープンソースでプラグイン対応のクロスプラットフォームIDE。IntelliJ IDEA（2001年）はJava向け高機能IDEとして登場しました。エンタープライズ向け大規模開発を支えるIDE群が出揃った時代です。

2000年代

Web開発ブーム・軽量エディタ

JavaScript, PHP, Rubyなどでブラウザ上アプリが主流に。Sublime Text（2008年）が重いIDEと簡素なメモ帳の間を埋め、「軽くて速いエディタ」のニーズを証明。

軽量化

Sublime Textは広告なし・開発者の口コミだけで普及した稀有なエディタ。C++/Python製でほぼ瞬時に起動し、シンタックスハイライト＋プラグイン拡張に対応。この時代に「重いIDE vs 軽いエディタ」の二極化が始まりました。

2010年代

VS Code・GitHub・クラウド開発

VS Code（2015年, Microsoft）が無料・オープンソースで登場し、世界で最も使われるコードエディタに。GitHubでのコラボレーションが当たり前の時代へ。

コラボの時代

VS Codeは拡張機能エコシステムの充実で、エディタとIDEの良いとこ取りを実現。GitHub Codespaces, Replitなど、ブラウザ上で完全な開発環境を提供するクラウドIDEも登場。Pull Request、コードレビュー、CI/CDの文化が確立されました。

2020年代〜

AIコーディングの時代

AIがコーディングのあり方を根本から変革。「人間がコードを書く」から「人間がAIに意図を伝え、AIがコードを書く」へのパラダイムシフトが進行中。

勉強会との接続

勉強会のデモ②（Gmail・Googleカレンダー連携）はMCPで実現しています！ Claudeがメールを読み、カレンダーに予定を追加するデモは、まさにMCPの実力です。

導入難易度

設定ファイル（JSON形式）の編集が必要。エンジニア向けですが、手順書があれば非エンジニアでも対応可能です。

MCPサーバーの設定は、以下のようなJSON形式のファイルで行います。

{
  "mcpServers": {
    "gmail": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-gmail"],
      "env": {
        "GMAIL_CLIENT_ID": "your-client-id",
        "GMAIL_CLIENT_SECRET": "your-secret"
      }
    },
    "google-calendar": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-google-calendar"]
    }
  }
}

このファイルを ~/.claude/settings.json に配置するだけで、ClaudeからGmailやカレンダーにアクセスできるようになります。

コードベースの高速検索・分析に特化した読み取り専用エージェント。ファイル検索やキーワード調査を素早く実行します。

Plan（設計）

実装計画を立てる設計専門エージェント。ファイル構成やアーキテクチャの検討を行い、ステップバイステップの計画を返します。

⚙️

General-purpose（汎用）

複雑で複数ステップのタスクを自律実行する汎用エージェント。ファイル操作・コマンド実行・Web検索など全ツールにアクセス可能。

カスタムサブエージェント 応用

自分専用のサブエージェントも作成できます。プロジェクト内の .claude/agents/ フォルダに定義ファイルを置くだけです。

Markdown + YAML形式で記述します。

---
name: security-checker
description: コードのセキュリティ脆弱性をチェックする
tools:
  - Read
  - Grep
  - Glob
---

あなたはセキュリティ専門のレビュアーです。
以下の観点でコードを分析してください：
- SQLインジェクション
- XSS（クロスサイトスクリプティング）
- 認証・認可の不備
- 機密情報のハードコード

この定義ファイルを .claude/agents/security-checker.md として保存すると、Claudeが自動的にセキュリティチェック時にこのエージェントを呼び出します。

メリット 利点

⚡

並列処理で高速化

最大7つのサブエージェントを同時実行可能。調査・実装・テストを並行して進め、作業時間を大幅に短縮。

🧠

コンテキストの節約

大量のログやテスト結果をサブエージェントが処理。親の会話が長大にならず、思考の質を維持できます。

🎯

専門性の向上

各エージェントに専門知識を与えることで、汎用的に聞くより精度の高い結果を得られます。

🔒

エラーの隔離

1つのサブエージェントが失敗しても他には影響しません。安全に試行錯誤できます。

♻️

再利用可能

一度作ったカスタムエージェントは他のプロジェクトでも再利用OK。ナレッジの資産化が可能。

📊

複雑なワークフロー

「調査→検証→レポート作成」のような多段階の業務フローを1つの指示で自動化できます。

活用イメージ 実例

📊 経営者の活用シーン

💼 業務効率化

  • コードレビュー：セキュリティ・品質・テストを3つのエージェントが同時チェック
  • データ分析：データ収集・分析・レポート作成を並列実行
  • Web制作：デザイン・コーディング・テストを分担

🔬 高度な活用

  • 市場調査：複数の情報源から並列で情報収集し、統合レポートを生成
  • 競合分析：各競合を別々のエージェントが調査し、比較表を自動作成
  • 決算分析：財務・外部環境・センチメントを3エージェントが並列調査

気をつけるべきポイント 注意

  • ⚠️
    親→子の情報共有は限定的
    サブエージェントは親の会話全体を見ることができません。必要な情報は指示の中で明確に伝える必要があります。
  • ⚠️
    複雑な相互依存には不向き
    「Aの結果を見てBを決める、Bの結果でCを変える」のような密結合タスクは、1つのエージェントで処理した方が効率的です。
  • ⚠️
    並列実行は最大7つまで
    同時に動かせるサブエージェントには上限があります。大規模なタスクは段階的に分ける必要があります。
  • ⚠️
    権限管理に注意
    サブエージェントがファイル操作やコマンドを実行する場合、権限の確認が必要です。意図しない操作を防ぐため、読み取り専用に制限する設計も有効です。
  • ⚠️
    結果の品質チェック
    サブエージェントの出力は必ず親が統合します。結果が不十分な場合は再度指示を出すか、手動で確認しましょう。
💡 経営者にとっての「サブエージェント」とは？
社長（親エージェント）が、経理・営業・法務の各部門長（サブエージェント）に仕事を振るイメージです。各部門長は専門知識を持ち、独立して作業し、結果だけを社長に報告します。部門間の仕事は同時並行で進むので、全体のスピードが劇的に上がります。
この資料でも活用

この別添え資料の制作でも、サブエージェントを活用しています。CLI・ターミナル・コーディングの歴史調査を調査エージェントが並列で情報収集し、メインエージェントが統合・執筆しています。

Claude CodeはNode.js上で動作します。まず Node.js をインストールします。

  1. https://nodejs.org/ にアクセス
  2. LTS版（推奨版）のダウンロードボタンをクリック
  3. ダウンロードしたインストーラーを実行
  4. 画面の指示に従ってインストール完了

確認コマンド：

node --version
# v20.x.x のように表示されればOK

Mac: 「アプリケーション」→「ユーティリティ」→「ターミナル」を開く。またはSpotlight（Cmd+Space）で「ターミナル」と検索。

Windows: スタートメニューから「PowerShell」または「コマンドプロンプト」を開く。

黒い画面（またはテーマ設定による色の画面）が表示されれば準備完了です。

ターミナルで以下のコマンドを実行します。

npm install -g @anthropic-ai/claude-code

しばらく待つと、インストールが完了します。エラーが出た場合は、Node.jsが正しくインストールされているか確認してください。

Macで権限エラーが出た場合は、先頭に sudo を付けて実行：

sudo npm install -g @anthropic-ai/claude-code

プロジェクトのディレクトリに移動してから起動します。

cd ~/my-project
claude

Claude Codeのインターフェースが表示されます。初回起動時はログインが求められます。

初回起動時にブラウザが開き、Anthropicアカウントでのログインを求められます。

  1. ブラウザで表示されるログイン画面でAnthropicアカウントを入力
  2. 認証が完了すると、ターミナルに「Authenticated!」と表示
  3. これでClaude Codeが使えるようになります

注意：Claude Code の利用には Pro以上のプラン（月額$20〜）が必要です。

全体の所要時間は約10〜15分。Node.jsさえインストールすれば、あとはコマンド1つでClaude Codeが使えます。

Node.jsプロジェクトのようですね。何かお手伝いしましょうか？

「command not found: claude」と表示される場合

  • Node.jsが正しくインストールされているか確認：node --version
  • npmのグローバルパスが通っているか確認：npm list -g
  • ターミナルを再起動してから再度試す

認証エラーが出る場合

  • Anthropicアカウントが有効か確認
  • Pro以上のプランに加入しているか確認
  • claude logout → claude で再ログイン

npmインストールでエラーが出る場合

  • Node.jsのバージョンが18以上か確認
  • Macの場合：sudo npm install -g @anthropic-ai/claude-code
  • Windowsの場合：PowerShellを「管理者として実行」
