📖 OVERVIEW

GitHub Pagesとは？

GitHubが提供する無料のWebホスティングサービスです。HTMLファイルをアップロードするだけで、世界中からアクセスできるWebページを公開できます。

🌐

無料で公開

サーバー費用ゼロ。GitHubアカウントがあれば誰でも無料で使えます

⚡

簡単3ステップ

難しい設定は不要。ファイルをアップするだけで公開完了

🔒

安全・安心

HTTPS対応済み。世界最大のソフトウェア開発プラットフォームが運営

必要なもの：メールアドレスだけ。クレジットカードの登録は不要です。

豆知識

GitHubは世界中で1億人以上が利用するソフトウェア開発プラットフォームです。2018年にMicrosoftが約8兆円で買収しました。GitHub Pagesを使えば、プログラミングの知識がなくてもWebサイトを公開できます。

👤
1️⃣ STEP 1

GitHubアカウント作成

まずはGitHubのアカウントを作りましょう。メールアドレスだけで登録できます。

1

github.com にアクセス

ブラウザ（Chrome、Safari等）で github.com を開きます。

2

「Sign up」をクリック

画面右上にある緑色の「Sign up」ボタンを押します。

3

メールアドレスを入力

普段お使いのメールアドレスを入力し、「Continue」を押します。

4

パスワードを設定

安全なパスワードを作成します。

5

ユーザー名を入力

英数字とハイフン（-）が使えます。他の人と重複しない名前を選んでください。

6

メール認証を完了

入力したメールアドレスに確認メールが届きます。メール内のリンクをクリックすれば完了です。

重要：ユーザー名は公開URLの一部になります。例えば、ユーザー名が tanaka-taro なら、公開URLは tanaka-taro.github.io になります。覚えやすい名前がおすすめです。

GitHubのパスワードは以下のいずれかの条件を満たす必要があります：

  • 15文字以上（どんな文字でもOK）
  • 8文字以上で、数字と小文字のアルファベットを含む

例：MyCompany2026!（12文字、数字+小文字+大文字+記号）

パスワードマネージャーの利用もおすすめです。

  • 会社名やブランド名を含めると分かりやすい（例：tanaka-corp）
  • 短くて覚えやすいものがおすすめ
  • 日本語は使えません（英数字とハイフンのみ）
  • あとから変更も可能ですが、URLも変わるので最初に決めるのがベストです
📁
2️⃣ STEP 2

リポジトリ作成

リポジトリは「ファイルを保管する場所」です。フォルダのようなものだと思ってください。

1

右上の「+」→「New repository」をクリック

GitHubにログインした状態で、画面右上の「＋」ボタンを押し、「New repository」を選びます。

2

Repository name を入力

リポジトリの名前を入力します。例：my-website

3

「Public」を選択

GitHub PagesはPublic（公開）リポジトリで無料で使えます。必ず「Public」を選んでください。

4

「Add a README file」にチェック

チェックを入れると、リポジトリの説明ファイルが自動的に作られます。あると便利なのでチェックしておきましょう。

5

「Create repository」をクリック

緑色のボタンを押せば、リポジトリの完成です。

重要：リポジトリ名もURLの一部になります。公開URLは次の形式になります。
https://ユーザー名.github.io/リポジトリ名/
  • 英数字とハイフン（-）を使いましょう
  • 短くてわかりやすい名前がおすすめ（例：company-site、product-page）
  • 日本語は使えません
  • スペース（空白）の代わりにハイフンを使います
📤
3️⃣ STEP 3

HTMLファイルのアップロード

作成したHTMLファイルをリポジトリにアップロードします。ドラッグ&ドロップで簡単にできます。

1

リポジトリページで「Add file」→「Upload files」

リポジトリのページで、緑色の「Add file」ボタンを押し、「Upload files」を選びます。

2

HTMLファイルをドラッグ&ドロップ

パソコン上のHTMLファイルを、表示されたエリアにドラッグ&ドロップします。または「choose your files」をクリックしてファイルを選択します。

3

ファイル名は index.html にする

アップロードするHTMLファイルの名前は必ず index.html にしてください。これがトップページとして認識されます。

4

「Commit changes」をクリック

ページ下部にある緑色の「Commit changes」ボタンを押せば、アップロード完了です。

重要：ファイル名は必ず index.html にしてください。これがWebサイトのトップページになります。大文字・小文字を間違えないようにしましょう（全て小文字です）。

画像ファイルやCSSファイルなど、複数のファイルを同時にアップロードできます。

  • 複数ファイルをまとめてドラッグ&ドロップすればOK
  • 画像は images フォルダを作って整理すると便利です
  • フォルダごとアップロードすることはできません。ファイルを個別に選択してください

「Commit（コミット）」は、ファイルの変更を保存・記録する操作のことです。パソコンでいう「保存」に似ていますが、変更履歴が残るのが特徴です。いつ、何を変えたかが記録されるので、あとで元に戻すこともできます。

⚙️
4️⃣ STEP 4

GitHub Pages有効化

アップロードしたHTMLを公開するための設定を行います。あと少しです！

1

リポジトリの「Settings」タブをクリック

リポジトリページの上部にあるタブ群から「Settings」（歯車アイコン）を選びます。

2

左メニューから「Pages」を選択

左側のメニューを下にスクロールし、「Pages」をクリックします。

3

「Source」→「Deploy from a branch」を選択

「Build and deployment」セクションの「Source」で「Deploy from a branch」を選びます。

4

「Branch」→「main」を選択

「Branch」のドロップダウンから「main」を選び、フォルダは「/ (root)」のままにします。

5

「Save」をクリック

「Save」ボタンを押せば、設定完了です。

注意：設定を保存してから、Webページが公開されるまで1〜3分かかります。すぐに表示されなくても心配しないでください。
豆知識

GitHub Pagesは裏側でHTMLファイルをCDN（コンテンツ配信ネットワーク）に配置しています。これにより、世界中どこからアクセスしても高速にページが表示されます。企業のWebサイトでも使われている仕組みです。

🎉
5️⃣ STEP 5

公開URLを確認

いよいよ最後のステップです。あなたのWebページが世界に公開されました！

1

Settings → Pages に戻る

先ほどと同じ「Settings」→「Pages」のページを開きます。

2

上部に公開URLが表示される

「Your site is live at ...」というメッセージとともに、公開URLが表示されます。

3

URLをクリックして確認

表示されたURLをクリックして、あなたのWebページが正しく表示されるか確認しましょう。

公開URLの形式

https://ユーザー名.github.io/リポジトリ名/
🎊
おめでとうございます！

あなたのWebページが世界に公開されました！

このURLを誰かに共有すれば、スマートフォンやパソコンからすぐにアクセスできます。
