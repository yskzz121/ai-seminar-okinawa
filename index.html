🖥️ CONTEXT WINDOW

コンテキストウィンドウの概念

Claudeの「作業デスク」を理解しよう

🗂️

📎 「作業デスク」の比喩

コンテキストウィンドウは、Claudeの「作業デスク」のようなもの。
デスクの上に広げた資料だけを見ながら仕事をしている——デスクに載っていない資料は「見えない＝存在しない」のと同じです。

📐

デスクが広い

一度に多くの資料を広げられる → 複雑な仕事ができる

📌

デスクが狭い

少しの資料しか置けない → 単純な質問向き

古い資料からデスクの端に落ちていく——会話が長くなるほど、冒頭の指示は薄れていきます。
トークンとは？

Claudeの現行モデルは約200,000トークンのコンテキストウィンドウを持ちます。「トークン」とは単語の断片のようなもの。日本語では1文字 ≒ 1〜2トークンが目安です。つまり、おおよそ10〜20万文字を一度に扱えます。

🔍 デスクの状態と実用的な意味

状況デスクの状態実用上の意味
短い質問1往復 デスクはガラ空き。余裕たっぷり 単発の質問・翻訳・要約に最適
長文資料を貼り付けて質問 デスク半分が資料で埋まる 資料の分析や要点抽出に向く
長い会話（数十往復） デスクが溢れかけている 初期の指示や文脈が押し出される危険あり
システムプロンプト＋資料＋長い会話 デスクの大半が占有済み 回答の質が落ちやすい。分割を検討
コンテキストウィンドウは有限資源です。「何をデスクに載せるか」を設計することが、AIの成果を左右します。

📏 情報の順番と指示の出し方 ― 3つの原則

1 重要な指示は「最初」と「最後」に置く

心理学の「初頭効果・親近効果」と同じ原理。中盤に埋もれた指示は見落とされやすい。

✅ 良い例：冒頭で「あなたは上間天ぷらの経営分析アシスタントです」→ 中盤にデータ → 末尾で「以上を踏まえ、週次KPIレポートを作成してください」
❌ 悪い例：データの途中に「ところでレポート形式で出力してね」と紛れ込ませる

2 否定指示は具体的な代替とセットで

「〜するな」だけでなく「代わりに〜してください」と書くと精度が上がります。

✅「箇条書きではなく、ですます調の文章で記述してください」
❌「箇条書きを使うな」（何で書けばいいか曖昧）

3 会話が長くなったら指示をリマインドする

20往復を超えるような長い会話では、重要な前提条件を途中で再提示しましょう。

✅「改めて確認ですが、出力は日本語のみ、ですます調、データ根拠を必ず添えてください」

⚙️ FEATURES

Claudeの主要機能

知っておくと活用の幅が広がる4つの機能

🔧
📋

仕様書（System Prompt）

Claudeの「人格設定書」。会話が始まる前にClaudeが必ず読み込む指示で、すべての回答のベースになります。

  • Claude.aiの「プロジェクト」機能で設定可能
  • コンテキストウィンドウの最上位に常駐
  • 例：「あなたは株式会社上間の経営企画担当です。回答はすべて日本語、ですます調で行ってください」
活用例

上間天ぷらの週次レポート作成プロジェクトでは、仕様書にブランドカラー（白・#CC2200・#F5C518）、KPI定義、出力フォーマットを定義。毎回指示し直す手間が省けます。

📄

マークダウンファイル

プロジェクトに添付できる参考資料。業務マニュアル、用語集、テンプレートなどを.mdファイルとしてアップロードすると、Claudeがその内容を踏まえて回答します。

  • コンテキストウィンドウの容量を消費する点に注意
  • 必要な情報だけを厳選して添付するのがコツ
活用例

「店舗別売上CSVのカラム定義書」をマークダウンで添付 → データ分析時にカラム名の意味を毎回説明する必要がなくなります。

🧠

メモリ機能

Claudeが会話をまたいで覚えておく情報のこと。ユーザーの名前、仕事の背景、好みの回答スタイルなどを蓄積し、次回以降の会話に自動的に反映されます。

  • 自動学習と、明示的に「覚えて」と指示するものがある
  • プライバシー配慮：設定からオフにすることも可能
  • 覚えた内容は「メモリの管理」画面で確認・削除可能
活用例

「私は沖縄のうるま市在住で、上間天ぷらの代表です」と一度伝えておけば、次回から前提説明なしに業務相談ができます。

🛠️

Skills（スキル）

Claude Codeが持つ「専門スキルセット」。Word、Excel、PowerPoint、PDFなどの作成・編集を、専用スキルに沿って高品質に実行できる仕組みです。

  • スキルはClaude内部のベストプラクティス集
  • ユーザーが意識しなくても自動で適用される
  • カスタムスキルの作成も可能

💬 PROMPT

話しかけ方の3つのコツ

プロンプト = Claudeへの「お願いの仕方」

1

具体的に伝える

「いい感じに」ではなく「箇条書きで3つ」

2

役割を与える

「あなたは経営コンサルタントです」

3

背景・目的を添える

「売上を伸ばしたい飲食店のため」

❌ 良くない例

売上を上げる方法を教えて

✅ 良い例

あなたは飲食店の経営コンサルタントです。沖縄県那覇市で沖縄料理の居酒屋を経営しています。平日ランチの集客を増やしたいです。予算5万円以内で実施できる施策を3つ、優先度順に提案してください。

  • 段階的に質問：一度に全部聞かず、対話で深掘り
  • 出力形式を指定：「表形式で」「箇条書きで」「〇〇文字以内で」
  • 例を示す：「例えばこんな感じで」と添える
  • ダメ出しOK：「もっとカジュアルに」「別の案を3つ」

🤔 ハルシネーションとは？

AIが「もっともらしいが事実と異なる情報」を生成してしまう現象。特に固有名詞・数値・歴史的事実で注意。

→ 重要な情報は必ず裏取り。AIの出力は「下書き」として扱おう。

事例1：売上データの分析を依頼する

❌ ありがちな指示 
このCSVを分析して
✅ 改善した指示 
添付の上間天ぷら週次売上CSV（2026年3月第2週）を分析してください。
・店舗別の売上前週比を算出

🔬 スコープ絞り込みのテクニック

📐 「5W1H＋制約」フレームワーク

WHO → 誰に向けた出力か？
WHAT → 何を作る／答えるのか？
WHY → 何のためか（目的・背景）？
WHEN → いつの情報・いつまでに？
WHERE → どこで使う（媒体・場面）？
HOW → どんな形式・スタイルで？
＋制約 → やってはいけないこと / 出力の長さ・言語・トーン
